## **核心决策树**

```
判断问题类型 → 选择排序方式
```

---

## **1. LeetCode 56 - 合并区间**
**问题**：合并所有重叠区间
**排序**：**按左边界排序**
**原因**：我们要按顺序处理区间，左边界排序后，相邻区间更容易判断是否重叠
```java
Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
// 然后判断：当前区间左边界 <= 上一个区间右边界 → 合并
```

**记忆口诀**：**"合并要左排，相邻好判断"**

---

## **2. LeetCode 435 - 无重叠区间** 
**问题**：移除最少数量的区间使剩余区间不重叠
**排序**：**按右边界排序**
**原因**：优先选择结束早的区间，给后面留更多空间
```java
Arrays.sort(intervals, (a, b) -> a[1] - b[1]);
// 贪心：总是选结束最早的，给后面留机会
```

**记忆口诀**：**"去重选右排，早结束多机会"**

---

## **3. LeetCode 452 - 用最少数量的箭引爆气球**
**问题**：用最少箭射爆所有气球（区间）
**排序**：**按右边界排序**
**原因**：一箭尽可能多射中结束早的气球
```java
Arrays.sort(points, (a, b) -> a[1] - b[1]);
// 箭放在当前区间右边界，能射爆所有左边界 <= 箭位置的气球
```

**记忆口诀**：**"射箭看右排，一箭多中的"**

---

## **4. LeetCode 253 - 会议室 II**
**问题**：最少需要多少会议室
**技巧**：**分开排序开始和结束时间**
```java
Arrays.sort(starts);
Arrays.sort(ends);
// 双指针：遇到开始时间 → 需要会议室，遇到结束时间 → 释放会议室
```

**记忆口诀**：**"会议室分开排，开始结束双指针"**

## **万能决策指南**

### **什么时候左边界排序？**
✅ **需要按顺序处理区间**（合并、划分等）
✅ **关注区间起始关系**
✅ 典型：56合并区间、763划分字母区间

### **什么时候右边界排序？**  
✅ **需要最大化数量/最小化移除**
✅ **关注"给后面留机会"**
✅ 典型：435无重叠区间、452射气球

### **什么时候分开排序？**
✅ **事件调度类问题**
✅ **有明确的开始和结束事件**
✅ 典型：253会议室II

| ---------- | -------- | ------------------ | -------- |
| 合并区间   | 左边界   | 相邻判断是否重叠   | 56       |

**终极记忆法**：
- **要合并** → 左排
- **要选择** → 右排  
- **要调度** → 分开排
