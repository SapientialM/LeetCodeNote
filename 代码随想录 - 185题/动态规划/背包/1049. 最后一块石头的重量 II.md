# [1049. 最后一块石头的重量 II](https://leetcode.cn/problems/last-stone-weight-ii/)

每次粉碎 min(x,y) 质量，因为要求剩余最小，那就约等于每次都尽量让 x、y相近。

```java
class Solution {
    public int lastStoneWeightII(int[] stones) {
        int sum = 0;
        for (int i : stones) {
            sum += i;
        }
        int target = sum >> 1;
        //初始化dp数组
        int[] dp = new int[target + 1];
        for (int i = 0; i < stones.length; i++) {
            //采用倒序
            for (int j = target; j >= stones[i]; j--) {
                //两种情况，要么放，要么不放
                dp[j] = Math.max(dp[j], dp[j - stones[i]] + stones[i]);
            }
        }
        return sum - 2 * dp[target];
    }
}
```

## 核心思路
**其实就是416题目的变体，将石头分成两堆，让两堆的重量尽可能接近，那么就是让其中一堆尽可能靠近总重量的一半，找到这一堆的大小 size，那其实结果就出来了：**

`result = (sum - size) - (sum - (sum - size)) = sum - 2*size`，也就是最后的结果。

## 为什么这样想？
- 每次选两块石头相撞，相当于从两堆中各取一块
- 如果两堆重量相等，最后剩下0
- 如果两堆重量接近，最后剩下的重量最小

## 代码解析

1. **计算总重量**：`sum` 是所有石头重量之和

2. **确定目标**：`target = sum / 2`（用位运算 `sum >> 1` 快速计算）

3. **0-1背包问题**：
   
   - 背包容量 = `target`
   - 物品重量 = 石头重量
   - 物品价值 = 石头重量
   - 目标：在不超过容量的情况下，尽可能装满背包
   
4. **动态规划**：
   ```java
   dp[j] = max(dp[j], dp[j - stones[i]] + stones[i])
   ```
   计算容量为j的背包能装的最大重量

5. **最终结果**：
   ```java
   return sum - 2 * dp[target]
   ```
   - `dp[target]` 是一堆的最大可能重量
   - 另一堆重量 = `sum - dp[target]`
   - 剩余石头 = 两堆重量差 = `sum - 2 * dp[target]`
